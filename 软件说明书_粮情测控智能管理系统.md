# 粮情测控智能管理系统软件说明书

## 1. 软件基本信息

### 1.1 软件名称
粮情测控智能管理系统

v1.0.0

### 1.3 开发单位
宁波思博达科技有限公司

## 2. 软件功能概述

粮情测控智能管理系统是一款专门用于粮食仓储企业的智能监控系统，通过传感器网络实时采集粮库内的温湿度数据，实现粮情的远程监控、数据分析和预警管理。系统支持多种采集设备类型，包括串口主机、网络主机和网络分机，为粮食仓储提供全面的智能化管理解决方案。

## 3. 系统架构

### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                            前端层                              │
│  Next.js 16 (App Router) + React 19 + Tailwind CSS + NextUI     │
├─────────────────────────────────────────────────────────────────┤
│                            API层                               │
│  Next.js API Routes + Server Actions                          │
├─────────────────────────────────────────────────────────────────┤
│                            服务层                              │
│  Prisma ORM + MQTT Client + 数据采集处理                      │
├─────────────────────────────────────────────────────────────────┤
│                            数据层                              │
│  PostgreSQL 数据库                                           │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 核心模块

1. **数据采集模块**：负责从各种采集设备获取温湿度数据
2. **数据存储模块**：将采集的数据存储到数据库
3. **用户管理模块**：处理用户认证、授权和权限管理
4. **粮库管理模块**：管理粮库和仓房信息
5. **数据分析模块**：对采集的数据进行分析和可视化
6. **MQTT通信模块**：处理与采集设备的MQTT通信

## 5. 系统流程

### 5.1 数据采集流程

1. **采集触发**：用户在前端发起采集请求
2. **参数配置**：系统根据仓房配置准备采集参数
3. **设备连接**：建立与采集设备的通信连接
4. **命令发送**：向设备发送采集命令
5. **数据接收**：接收设备返回的采集数据
6. **数据解析**：解析原始数据为结构化格式
7. **数据存储**：将解析后的数据存储到数据库
8. **状态更新**：更新仓房采集状态
9. **结果反馈**：向前端返回采集结果

### 5.2 用户操作流程

1. **用户登录**：用户通过用户名密码登录系统
2. **权限验证**：系统验证用户权限
3. **功能访问**：用户访问相应功能模块
4. **操作执行**：用户执行具体操作（如采集、配置等）
5. **结果反馈**：系统返回操作结果

## 8. 系统配置与部署

### 8.1 环境配置

系统通过.env文件进行配置管理，主要配置项包括：

- DATABASE_URL：数据库连接字符串
- NEXTAUTH_SECRET：认证密钥
- MQTT_BROKER_URL：MQTT broker地址
- MQTT_USERNAME：MQTT用户名
- MQTT_PASSWORD：MQTT密码

### 8.2 部署步骤

1. **环境准备**：安装Node.js 18+和PostgreSQL
2. **依赖安装**：运行 `npm install`
3. **数据库迁移**：运行 `npm run db:push`
4. **数据初始化**：运行 `npm run db:seed`
5. **构建项目**：运行 `npm run build`
6. **启动服务**：运行 `npm start`
